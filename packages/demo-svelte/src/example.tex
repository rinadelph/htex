\section{Introduction to Reinforcement Learning}

Reinforcement learning (RL) is a paradigm for learning sequential decision-making policies through interaction with an environment. Unlike supervised learning, which requires labeled training examples, RL agents learn from rewards or penalties received as feedback for their actions.

\subsection{Core Concepts}

\subsubsection{Markov Decision Processes}

A Markov Decision Process (MDP) is a formal mathematical framework for modeling sequential decision-making problems. An MDP is defined by the tuple:

\[
\mathcal{M} = \langle \mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R}, \gamma, \rho_0 \rangle
\]

where:
\begin{itemize}
    \item $\mathcal{S}$ is the state space
    \item $\mathcal{A}$ is the action space
    \item $\mathcal{T}: \mathcal{S} \times \mathcal{A} \times \mathcal{S} \to [0,1]$ is the transition model
    \item $\mathcal{R}: \mathcal{S} \times \mathcal{A} \to \RR$ is the reward function
    \item $\gamma \in [0,1]$ is the discount factor
    \item $\rho_0: \mathcal{S} \to [0,1]$ is the initial state distribution
\end{itemize}

\subsubsection{Policies and Value Functions}

A policy $\pi: \mathcal{S} \times \mathcal{A} \to [0,1]$ is a mapping from states to action distributions. The value function $V^{\pi}(s)$ represents the expected cumulative discounted reward obtained by following policy $\pi$ starting from state $s$:

\[
V^{\pi}(s) = \EE_{\pi}\left[\sum_{t=0}^{\infty} \gamma^t r_t \given s_0 = s\right]
\]

Similarly, the action-value function (Q-function) is defined as:

\[
Q^{\pi}(s,a) = \EE_{\pi}\left[\sum_{t=0}^{\infty} \gamma^t r_t \given s_0 = s, a_0 = a\right]
\]

\subsection{Value-Based Methods}

Value-based reinforcement learning methods learn an approximation of the optimal value function:

\[
V^*(s) = \max_{\pi} V^{\pi}(s)
\]

The optimal policy can then be recovered via:

\[
\pi^*(s) = \argmax_{a \in \mathcal{A}} Q^*(s,a)
\]

\subsubsection{Temporal Difference Learning}

Temporal Difference (TD) learning combines ideas from dynamic programming and Monte Carlo methods. The TD(0) update rule for the value function is:

\[
V(s) \leftarrow V(s) + \alpha \left[r + \gamma V(s') - V(s)\right]
\]

where $\alpha$ is the learning rate, $r$ is the immediate reward, and $s'$ is the next state.

\subsubsection{Q-Learning}

Q-learning is an off-policy TD method that learns the optimal action-value function. The update rule is:

\[
Q(s,a) \leftarrow Q(s,a) + \alpha \left[r + \gamma \max_{a'} Q(s',a') - Q(s,a)\right]
\]

One key advantage of Q-learning is that it does not require the agent to follow the learned policy during training, making it an off-policy algorithm.

\section{Deep Reinforcement Learning}

Deep reinforcement learning combines neural networks with RL algorithms to handle high-dimensional state and action spaces. 

\subsection{Deep Q-Networks (DQN)}

Deep Q-Networks (DQN) use a neural network to approximate the Q-function:

\[
Q(s,a;\theta) \approx Q^*(s,a)
\]

The DQN algorithm maintains two networks:
\begin{enumerate}
    \item Main network: $Q(s,a;\theta)$ updated on every step
    \item Target network: $Q(s,a;\theta^-)$ updated less frequently
\end{enumerate}

The TD target uses the target network to stabilize training:

\[
y_i = r_i + \gamma \max_{a'} Q(s_i', a'; \theta^-)
\]

The loss function minimizes the Mean Squared Error (MSE):

\[
\LL(\theta) = \EE\left[\left(y_i - Q(s_i, a_i; \theta)\right)^2\right]
\]

\subsection{Policy Gradient Methods}

\subsubsection{REINFORCE}

The REINFORCE algorithm uses the policy gradient theorem to directly optimize the policy. The gradient estimate is:

\[
\nabla J(\theta) = \EE_{\pi_\theta}\left[\nabla \log \pi_\theta(a \given s) Q^{\pi_\theta}(s,a)\right]
\]

The update rule is:

\[
\theta \leftarrow \theta + \alpha \nabla \log \pi_\theta(a \given s) G_t
\]

where $G_t = \sum_{k=t}^{T} \gamma^{k-t} r_k$ is the return.

\subsubsection{Actor-Critic Methods}

Actor-Critic methods combine policy gradients (actor) with value function estimation (critic):

\[
\nabla J(\theta) = \EE_{\pi_\theta}\left[\nabla \log \pi_\theta(a \given s) \cdot A^{\pi}(s,a)\right]
\]

where $A^{\pi}(s,a) = Q^{\pi}(s,a) - V^{\pi}(s)$ is the advantage function, which can be estimated as:

\[
A^{\pi}(s,a) \approx r + \gamma V(s') - V(s)
\]

\subsection{Proximal Policy Optimization (PPO)}

PPO is a modern policy gradient method that uses a clipped surrogate objective to prevent large policy changes:

\[
L^{CLIP}(\theta) = \EE_t\left[\min(r_t(\theta) \hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t)\right]
\]

where $r_t(\theta) = \frac{\pi_\theta(a_t \given s_t)}{\pi_{\theta_{\text{old}}}(a_t \given s_t)}$ is the probability ratio and $\hat{A}_t$ is the estimated advantage.

\subsubsection{Nash Equilibrium}

In competitive settings, agents learn towards Nash Equilibrium, where no agent can improve its payoff by unilaterally changing its strategy:

\[
\pi_i^* \in \argmax_{\pi_i} \EE[R_i \given \pi_i, \pi_{-i}^*]
\]

\subsection{Model-Based Reinforcement Learning}

Model-based methods learn a dynamics model $\hat{\mathcal{T}}$ of the environment and use it for planning:

\[
\hat{\mathcal{T}}_{\theta}(s' \given s, a) \approx \mathcal{T}(s' \given s, a)
\]

The planning procedure uses the learned model to imagine trajectories without real environment interaction:

\begin{align}
\hat{s}_0 &= s_{\text{real}} \\
\hat{a}_t &\sim \pi(\hat{a}_t \given \hat{s}_t) \\
\hat{s}_{t+1} &\sim \hat{\mathcal{T}}(\hat{s}_{t+1} \given \hat{s}_t, \hat{a}_t) \\
\hat{r}_t &= \mathcal{R}(\hat{s}_t, \hat{a}_t)
\end{align}

\subsection{Offline Reinforcement Learning}

Offline RL learns from a fixed dataset without environment interaction. A key concern is Q-learning overestimation. Conservative Q-Learning (CQL) adds a penalty term:

\[
\LL_Q(\theta) = \EE_{(s,a) \sim \mathcal{D}}\left[\left(r + \gamma \max_{a'} Q(s',a';\theta^-) - Q(s,a;\theta)\right)^2\right] - \alpha \EE_{s \sim \mathcal{D}}\left[\max_a Q(s,a;\theta)\right]
\]

\section{Integration with Large Language Models}

Large Language Models (LLMs) have been successfully integrated with RL in several ways:

\subsection{LLM as Policy}

LLMs can be used to directly generate actions. The policy is conditioned on the current state:

\[
\pi_{\text{LLM}}(a \given s) = p_{\text{LLM}}(\text{action tokens} \given \text{state description})
\]

\subsection{LLM for Reward Specification}

Natural language descriptions of desired behavior can be used to define reward functions:

\[
r(s,a) = f_{\text{LLM}}(\text{behavior description}, s, a)
\]

\section{Implementation Considerations}

\subsection{Experience Replay}

Experience replay stores transitions in a buffer and samples minibatches uniformly for training. This breaks temporal correlations and improves sample efficiency:

\begin{lstlisting}[language=Python]
class ReplayBuffer:
    def __init__(self, capacity):
        self.buffer = []
        self.capacity = capacity
    
    def push(self, state, action, reward, next_state, done):
        if len(self.buffer) >= self.capacity:
            self.buffer.pop(0)
        self.buffer.append((state, action, reward, next_state, done))
    
    def sample(self, batch_size):
        indices = random.sample(range(len(self.buffer)), batch_size)
        return [self.buffer[i] for i in indices]
\end{lstlisting}

\subsection{Target Network Updates}

The target network $\theta^-$ is periodically synchronized with the main network:

\[
\theta^- \leftarrow \tau \theta + (1 - \tau) \theta^-
\]

where $\tau$ is typically very small (e.g., $\tau = 0.001$).

\section{Challenges and Future Directions}

\subsection{Sample Efficiency}

While RL algorithms are powerful, they often require millions of environment interactions. Methods to improve sample efficiency include:

\begin{itemize}
    \item Model-based methods that use learned dynamics models
    \item Imitation learning from expert demonstrations
    \item Transfer learning from related tasks
    \item Curriculum learning with gradually increasing task difficulty
\end{itemize}

\subsection{Exploration-Exploitation Tradeoff}

Balancing exploration (trying new actions) and exploitation (maximizing known rewards) is fundamental to RL:

\[
\pi_{\text{balanced}}(a \given s) = (1-\epsilon) \argmax_a Q(s,a) + \epsilon \frac{1}{|\mathcal{A}|}
\]

Alternative exploration strategies include:
\begin{enumerate}
    \item Epsilon-greedy: Choose random action with probability $\epsilon$
    \item Softmax: $\pi(a \given s) \propto e^{Q(s,a) / \tau}$
    \item Upper Confidence Bound (UCB): Select actions with highest optimism
    \item Thompson Sampling: Sample from posterior distribution of value functions
\end{enumerate}

\subsection{Function Approximation}

When state/action spaces are large or continuous, function approximation is necessary. However, this introduces challenges:

\section{Conclusion}

Reinforcement learning is a powerful paradigm for learning sequential decision-making policies. From classic value-based methods like Q-learning to modern policy gradient methods like PPO, the field has developed a rich collection of algorithms. Recent integration with large language models opens new possibilities for combining symbolic reasoning with learning.

Key takeaways:
\begin{enumerate}
    \item Value-based methods optimize expected cumulative reward
    \item Policy gradient methods directly optimize the policy
    \item Model-based methods leverage learned environment models
    \item Multi-agent and offline settings present unique challenges
    \item LLM integration enables new applications and capabilities
\end{enumerate}

\section{Mathematical Background}

\subsection{Probability and Statistics}

For a random variable $X$, the expectation is:
\[
\EE[X] = \sum_x x \cdot \prob{X=x}
\]

The variance is:
\[
\VarOp(X) = \EE[(X - \EE[X])^2] = \EE[X^2] - (\EE[X])^2
\]

\subsection{Linear Algebra}

For vectors $\mathbf{u}, \mathbf{v} \in \RR^n$, the dot product is:
\[
\langle \mathbf{u}, \mathbf{v} \rangle = \sum_{i=1}^n u_i v_i
\]

The matrix-vector product $\mathbf{y} = \mathbf{A}\mathbf{x}$ where $\mathbf{A} \in \RR^{m \times n}$ and $\mathbf{x} \in \RR^n$ produces $\mathbf{y} \in \RR^m$:
\[
y_i = \sum_{j=1}^n A_{ij} x_j
\]

\subsection{Calculus}

The gradient of a function $f: \RR^n \to \RR$ is:
\[
\nabla f(\mathbf{x}) = \left[\frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \ldots, \frac{\partial f}{\partial x_n}\right]^\top
\]

The chain rule for composition $h(x) = f(g(x))$ gives:
\[
\frac{dh}{dx} = \frac{df}{dg} \cdot \frac{dg}{dx}
\]
